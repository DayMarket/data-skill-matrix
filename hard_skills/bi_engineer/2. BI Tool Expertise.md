# 2. BI Tool Expertise

Этот трек описывает глубину владения основным BI-инструментом (Superset). Рост в этом треке означает переход от базового создания дашбордов к продвинутой настройке, использованию динамических запросов, управлению доступами и, в конечном итоге, к расширению возможностей самого инструмента.

## BI1 (Junior)
- Умеет создавать и использовать основные сущности инструмента: графики, фильтры, дашборды.
- Ориентируется в интерфейсе BI-инструмента и понимает базовую функциональность.
- Следует установленным паттернам при создании дашбордов.

## BI2 (Middle)
- Умеет конфигурировать продвинутые функции: аннотации, рассылки, алерты.
- Настраивает свойства датасетов с вниманием к осям, легенде и форматированию.
- Может найти и устранить ошибку в любом отчёте (расхождение данных, проблемы загрузки).
- Самостоятельно создаёт и управляет датасетами.

## BI3 (Middle+)
- Использует аннотации для визуализации причинно-следственных связей и ключевых событий.
- Умеет управлять безопасностью доступов с помощью RBAC и RLS (Row-Level Security).
- Проводит тренинги и консультации для других аналитиков.
- Использует Jinja-шаблоны для создания динамических параметризованных запросов.
- Применяет CSS для кастомизации внешнего вида дашбордов.

## BI4 (Senior)
- Проектирует модели доступов и governance-политики на уровне всей организации.
- Создаёт переиспользуемые CSS-библиотеки стилей, которые применяются в дашбордах других команд.
- Модифицирует и расширяет кастомные плагины визуализаций.
- Создаёт автоматизированные alert-системы на отклонение ключевых метрик.
- Документирует ограничения платформы и формулирует запросы на улучшения.

## BI5 (Staff)
- Разрабатывает кастомные плагины визуализаций с нуля.
- Определяет архитектуру платформы и стратегии интеграций.
- Руководит оценкой и внедрением новых функций и версий BI-инструмента.
