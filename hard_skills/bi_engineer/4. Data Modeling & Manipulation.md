# 4. Data Modeling & Manipulation

Этот трек охватывает технические навыки по подготовке данных для аналитики: SQL, архитектура данных, разработка ETL и оптимизация производительности для построения надёжного фундамента данных.

## BI1 (Junior)
- Пишет корректные SQL-запросы (SELECT, JOIN, UNION, GROUP BY) с читаемым форматированием и именованием.
- Проверяет запросы через EXPLAIN, избегает full scan, исправляет замечания ревьюера.

## BI2 (Middle)
- Использует CTE, оконные функции, массивы ClickHouse, выбирая подходящий тип данных.
- Читает план запроса и переписывает его (фильтры, предагрегация, тип JOIN) с ускорением минимум в 2 раза.
- Умеет настраивать ETL и создавать витрины с помощью Airflow по шаблонам коллег.
- Анализирует требования для доработок хранилища.
- Скорость загрузки дашбордов не превышает 15 секунд.
- Создаёт корректные витрины данных без дублей.
- Использует dbt для трансформации данных и применяет проверки качества данных (DQ).
- Создаёт ETL-пайплайны, включая этапы интеграции и очистки данных.
- Пишет Python-скрипты автоматизации по шаблонам, создаёт алерты на ошибки ETL, применяет сенсоры в DAG.

## BI3 (Middle+)
- Проектирует end-to-end архитектуру данных для новых аналитических продуктов (raw → ... → mart).
- Проектирует витрины с оптимальным DDL (ORDER BY, типы данных, партиции).
- Делает peer-review сложных SQL, устраняет N+1 и дубли.
- Использует партиции для инкрементального обновления данных, понимает trade-offs между инкрементальным и полным обновлением.
- Делает стабильную и быстро загружаемую отчётность (ошибки у не более 10% уникальных пользователей).
- Оптимизирует сложные запросы разными способами: изменение структуры хранения, отчётности или запроса.
- Проактивно вносит исправления в витрины и датасеты, на которых строится отчётность.
- Понимает логику ETL от Data Engineers, разбирается в чужих Python-скриптах автоматизации.

## BI4 (Senior)
- Ведёт пост-мортемы медленных запросов, пишет гайды, организует воркшопы «Advanced SQL».
- Оптимизирует схемы хранения данных с использованием проекций, индексов и продвинутых возможностей БД.
- Проактивно мониторит потребление данных дашбордами.
- Проектирует ETL с применением временных таблиц для сложных трансформаций.
- Скорость загрузки дашбордов не превышает 10 секунд.
- Работает с API внешних сервисов для загрузки данных и интеграций.
- Пишет production-quality Python-скрипты (рассылки, автоматизация), оптимизирует скрипты коллег.
- Находит и предлагает системные подходы для оптимизации работы с хранилищем (мониторинг запросов, лимиты ресурсов, автоматические проверки SQL).

## BI5 (Staff)
- Признанный эксперт по нескольким бизнес-доменам и их моделям данных.
- Руководит инициативами по моделированию хранилища данных и архитектурными решениями.
- Проектирует и внедряет системные подходы для оптимизации работы с хранилищем (мониторинг запросов, управление ресурсами, автоматизированная валидация SQL).
