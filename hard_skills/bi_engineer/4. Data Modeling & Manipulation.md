# 4. Data Modeling & Manipulation

Этот трек охватывает технические навыки по подготовке данных для аналитики: SQL, архитектура данных, разработка ETL и оптимизация производительности для построения надёжного фундамента данных.

## BI1 (Junior)
- Пишет корректный и читаемый SQL (SELECT, JOIN, UNION, GROUP BY).
- Проверяет корректность результатов
- Понимает разницу между OLAP и OLTP.

## BI2 (Middle)
- Быстро пишет корректный и читаемый SQL.
- Понимает архитектуры Star и Snowflake.
- Использует CTE, оконные функции, массивы ClickHouse, выбирая подходящий тип данных
- Оптимизирует запросы фильтрацией и предагрегацией.
- Умеет создавать ETL, начиная с этапа интеграции данных и их очистки
- Создаёт простые витрины и DAG-и по готовым шаблонам
- Создает корректные витрины данных без дублей
- Работает с dbt на уровне моделей (создание и обновление таблиц)
- Применяет DQ-чек-листы для проверки данных.
- Успешно анализирует проблематику для доработок хранилища
- Скорость загрузки дашбордов не превышает 15 секунд
- Пишет свои скрипты по автоматизации Python, используя шаблоны и примеры
- Создает алерты на ошибки в ETL
- Применяет сенсоры в дагах
- Умеет читать чужой код (ETL от DE, скрипты Python)
- Верхнеуровнево знает витрины и источники по ключевым доменам (Маркетинг, Коммерция, Логистика, Финансы, Операции), понимает отличия и ограничения. 
- Составляет Source-of-Truth map для метрик (какой источник использовать и почему).

## BI3 (Middle+)
- Проектирует витрины с оптимальной структурой (ORDER BY, PARTITION BY, типы данных).
- Проектирует архитектуру хранения данных фичей (end‑to‑end пайплайн под новый аналитический продукт: raw →...→ mart)
- Знает SCD2/SCD3 подходы.
- Делает peer‑review сложных SQL, устраняет N+1 и дубли
- Использует партиции для обновления данных
- Настраивает инкрементальные и полные обновления.
- Используется временные или промежуточные таблицы.
- Знает детали базы данных и умеет оптимизировать сложные запросы различными способами - изменяя структуру хранения, отчетность, или запрос
- Делает SQL-ревью коллег (находит дубли, неоптимальные join-ы, медленные подзапросы).
- Обеспечивает стабильность и скорость отчётности: стабильная загрузка <10 сек, Fail Rate < 5%, Уникальных пользователей с ошибками не должно превышать 10%
- Визуализирует поток данных как mindmap: от источника до отчёта
- Находит и предлагает решения/системные подходы, которые оптимизируют работу с хранилищем
- Глубоко погружен в модель данных одного из доменов (Маркетинг, Коммерция, Логистика, Финансы, Операции). Знает полный lineage данных одного из доменов.

## BI4 (Senior)
- Проектирует архитектуру хранения данных сервисов (end‑to‑end пайплайн под новый аналитический продукт: raw →...→ mart)
- Моделирует агрегаты в хранилище, которые имеют зависимости от нескольких команд
- Ведёт пост‑мортемы медленных запросов, пишет гайды, организует воркшопы «Advanced SQL».
- Умеет оптимизировать схему хранения данных, используя проекции, индексы и прочие вещи, если понадобится
- Проектирует ETL, применяя временные таблицы
- Скорость загрузки дашбордов не превышает 10 секунд
- Пишет оптимальные скрипты по автоматизации Python (рассылка, автоматизация рутинных задач), которые выполняются на регулярной основе
- Оптимизирует ETL, Python и SQL коллег
- Внедряет решения/системные подходы, которые оптимизируют работу с хранилищем на уровне команды  
- Является экспертом в данных нескольких крупных доменах.

## BI5 (Staff)
- Курирует развитие корпоративного DWH и стандартов моделирования
- Задает стандарты работы, ведя организацию к более качественному хранилищу
- Понимает и моделирует данные по нескольким ключевым доменам компании (Коммерция, Маркетинг, Логистика, Финансы, Операции).
- Может описать value stream каждого процесса, знает источники данных и метрики.
- Курирует согласование бизнес-метрик между командами и обеспечивает целостность данных в кросс-доменных витринах
- Внедряет решения/системные подходы, которые оптимизируют работу с хранилищем на уровне организации
